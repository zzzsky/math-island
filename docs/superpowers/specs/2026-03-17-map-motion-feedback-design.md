# 数学岛地图动效反馈设计

## 1. 概述

本设计是在现有地图/岛屿交互体验基础上的一次轻量增强，目标是在不改动课程状态机、解锁规则和持久化结构的前提下，补齐地图回流时的空间反馈、数值反馈和奖励反馈。

本轮只做短时、可打断的 Compose 动效，不引入新的动画框架，不接正式美术资源。

## 2. 设计目标

- 新岛解锁时，孩子能在地图上立即感知“新路线”和“新目标岛”。
- 星星增加时，顶部统计不再硬切，改成短时跳变反馈。
- 宝箱有新收藏时，按钮出现轻量强调，提示玩家去查看。
- 所有反馈都只播放一次，不影响后续自由操作。

## 3. 非目标

- 不做长序列剧情动画。
- 不引入 Lottie、Spine 或音效系统。
- 不修改通关、评分、复习或宝箱解锁业务规则。
- 不把动画状态写回持久化。

## 4. 状态模型

在地图 UI 层引入一次性的 `MapMotionState`，由当前 `MapFeedbackUiState` 投影得到。

建议字段：

- `highlightedIslandId`
  新岛解锁时需要高亮的岛屿。
- `starsDelta`
  本次回到地图新增的星星数量。
- `chestPulse`
  是否对宝箱按钮播放短时强调。
- `active`
  当前动效是否仍在播放。

该状态只存在于 `MapTabletScreen` 内部，不进入业务进度。

## 5. 组件边界

### 5.1 IslandMapCanvas

负责：

- 目标岛节点脉冲
- 新路线高亮
- 已有选中态和高亮态的合并表达

不负责：

- 顶部星星跳变
- 宝箱按钮强调
- 反馈消费

### 5.2 MapProgressFeedback

负责：

- 顶部反馈文案
- 星星 pill 的强调表现
- 宝箱 pill 的可视提示

不负责：

- 地图节点绘制
- 宝箱按钮动画

### 5.3 MapTabletScreen

负责：

- 从 `feedback` 生成一次性 `MapMotionState`
- 协调画布、顶部星星和宝箱按钮三类反馈
- 在反馈完成后消费 `feedback`

## 6. 动效规则

### 6.1 新岛解锁

- 目标岛外圈在约 `700ms` 内做一次放大回落
- 新路线在约 `500ms` 内做一次亮度增强
- 若用户在播放期间点击其他岛，立即中止本轮动效

### 6.2 星星增长

- 顶部 `总星星` 从旧值补间到新值
- 持续约 `600ms`
- 同时星星 pill 做一次轻量发光或放大

### 6.3 宝箱提示

- `打开宝箱` 按钮做 `2-3` 次轻量呼吸或描边闪动
- 总时长不超过 `900ms`
- 不做无限循环

## 7. 时序

1. 地图进入 `MAP`
2. `MapTabletScreen` 从 `feedback` 派生 `MapMotionState`
3. 同步启动：
   - 目标岛/航线反馈
   - 星星统计反馈
   - 宝箱按钮反馈
4. 动效完成或被用户打断
5. 调用 `onConsumeFeedback()`

## 8. 测试策略

### 8.1 组件级

- `IslandMapCanvasTest`
  - 高亮岛存在时展示节点和航线反馈
- `MapProgressFeedbackTest`
  - 星星增量和宝箱提示正确显示

### 8.2 屏幕级

- `MapTabletScreenTest`
  - 动效态下宝箱按钮仍可见
  - 反馈节点高亮正确渲染
  - 反馈消费后再次进入不重复展示

### 8.3 设备流

- `MathIslandTabletFlowTest`
  - 通关回地图后，出现新岛高亮、星星变化和宝箱提示

## 9. 验收标准

- 新岛解锁时地图上可见明显但轻量的节点/航线反馈
- 顶部星星统计具有短时变化反馈
- 宝箱按钮在有新收藏时出现短时强调
- 所有反馈都不阻塞交互，且只消费一次
