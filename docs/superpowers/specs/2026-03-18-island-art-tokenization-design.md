# 数学岛地图本体美术插槽与视觉 Token 设计

## 1. 概述

本设计聚焦地图上的岛屿与航线本体，不改动课程状态机、解锁规则、地图交互合同和测试标签，只对地图绘制层做手绘冒险感增强，并为后续正式美术资源接入预留明确插槽与规格。

本轮目标不是直接接入最终插画，而是建立一套稳定的地图视觉语言、资源命名、资源尺寸和 fallback 机制。这样当前版本可以继续使用 Compose painter 产出可玩的默认样式，后续美术资源到位后可以逐步替换，而不需要重写地图交互结构。

## 2. 设计目标

- 地图的岛屿和航线具备统一的手绘冒险感，而不是普通节点图。
- 所有未来需要接美术资源的位置都先固定成明确插槽。
- 岛屿、海面、航线、状态叠层都支持 `asset or fallback painter`。
- 保持现有交互和稳定测试合同不变，尤其是地图节点、航线高亮和岛屿入口标签。

## 3. 非目标

- 不引入正式插画资产或外部动画框架。
- 不改变地图布局、点击逻辑、焦点切换和课程入口规则。
- 不把视觉资源选择写入业务状态。
- 不在本轮重做右侧面板、课程页或奖励页。

## 4. 视觉方向

本轮采用“轻手绘冒险感”：

- 海面：浅蓝到青绿色的纸感海面，带极轻波纹和云影层。
- 岛屿：从普通节点改成绘本式岛牌，带不规则轮廓、浅描边和柔和投影。
- 航线：从普通连线改成手绘虚线或柔和曲线。
- 状态：`locked / unlocked / focused / completed` 继续保留，用色块、描边、贴纸徽章和高亮外圈表达。

这个方向保留当前代码结构和测试边界，同时为后续插画升级留出余地。

## 5. 美术插槽

### 5.1 海面背景

- 资源 key：`map_sea_backdrop`
- 用途：整张地图海面底图、浅波纹、云影
- 推荐格式：PNG 或 WebP
- 推荐原稿尺寸：`1920x1200`
- 约束：
  - 允许有纸感纹理和波纹
  - 不承载文字
  - 亮度保持低干扰，不能压过岛屿本体

### 5.2 岛屿主图

每座岛预留一张底牌主图：

- 资源 key：`island_<islandId>_base`
- 用途：岛屿轮廓、地貌、装饰
- 推荐格式：透明背景 PNG 或 WebP
- 推荐原稿尺寸：`640x480`
- 约束：
  - 安全内容区按 `4:3` 控制
  - 主图不包含状态文案
  - 需适配小尺寸缩放后的可读性

### 5.3 状态叠层

- `island_locked_overlay`
- `island_focused_ring`
- `island_completed_badge`

规格：

- 推荐格式：透明 PNG 或 vector
- 基准尺寸：`256x256`
- 用途：
  - `locked`：灰蓝低饱和遮罩或雾层
  - `focused`：蜡笔感聚焦描边或手绘光圈
  - `completed`：右上角贴纸式完成徽章

约束：

- 不承载文字
- 必须允许代码继续绘制岛屿标题
- 可独立叠加在任何岛屿主图之上

### 5.4 航线贴图

- `route_segment_default`
- `route_segment_highlight`

规格：

- 推荐格式：透明 PNG 或 vector
- 单段基准长边：`512`
- 用途：
  - 默认航线的手绘虚线或柔和曲线纹理
  - 解锁高亮时的亮线扫光或暖色描边

约束：

- 允许平铺或重复拉伸
- 不承载固定方向箭头
- 需要兼容当前不同岛之间的连线长度

### 5.5 地图装饰物

- `map_cloud_*`
- `map_wave_*`
- `map_spark_*`

规格：

- 推荐格式：透明 PNG 或 vector
- 尺寸范围：`128~512`
- 用途：增强纸感海面和地图空间氛围

约束：

- 装饰物只做点缀，不能遮挡岛屿点击区域
- 装饰资源必须允许关闭或缺省，不影响主流程

### 5.6 岛屿小图标

- 资源 key：`island_<islandId>_icon`
- 推荐尺寸：`96x96`
- 推荐格式：vector 或透明 PNG
- 用途：小尺寸标签区、未来的面板或成就区复用

## 6. 代码结构

### 6.1 MapIllustrationTokens

新增地图专用 token 文件，用于集中定义：

- 岛屿主图尺寸
- 航线样式
- 海面渐变和 fallback 颜色
- 状态叠层尺寸
- 资源 key 常量

这套 token 不放进通用 theme，避免地图插画语义污染全局 UI 主题。

### 6.2 MapArtRegistry

新增资源注册层，负责：

- 按 key 解析海面、岛屿、叠层、航线和装饰资源
- 在没有正式资源时返回 fallback painter 方案
- 给地图绘制层提供统一入口

该层只负责资源选择，不负责点击逻辑、解锁逻辑或布局。

### 6.3 Painter 层

地图画布拆成三类 painter：

- `SeaBackdropPainter`
- `IslandNodePainter`
- `RoutePainter`

这三层都支持：

- 先尝试读取资源插槽
- 若资源不存在，则使用 Compose fallback painter

### 6.4 IslandMapCanvas

`IslandMapCanvas` 保留地图交互和稳定测试标签，但不再硬编码全部形状细节，只负责编排：

- 背景层
- 航线层
- 岛屿层
- 状态叠层
- 高亮与反馈层

## 7. 测试与合同

本轮必须保持以下合同不变：

- `map-node-<islandId>`
- `map-node-highlight-<islandId>`
- `map-route-highlight-<islandId>`
- `panel-start-<lessonId>`

需要补的测试：

- `IslandMapCanvasTest`
  - fallback painter 仍可渲染可点击岛屿
  - 指定高亮岛时，节点和航线标签仍存在
  - 无正式资源时，画布仍能正常绘制
- `MapTabletScreenTest`
  - 地图交互合同不因 painter 分层而变化

## 8. 实施顺序

1. 先建立 `MapIllustrationTokens` 与 `MapArtRegistry`
2. 再将 `IslandMapCanvas` 拆到海面/岛屿/航线 painter 层
3. 最后替换当前地图本体默认外观，做成轻手绘冒险感

## 9. 验收标准

- 地图本体视觉明确从“普通节点图”提升为“轻手绘冒险地图”
- 岛屿、航线、海面、状态叠层都有明确的未来资源插槽
- 所有资源规格可直接提供给美术同学使用
- 当前交互和主测试合同不回归
