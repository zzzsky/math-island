# Math Island 横版平板网页原型设计文档

## 1. 概述

本设计文档定义 `Math Island` 的第一版网页原型，用于在浏览器中预览横版平板主流程，并确认产品主体风格。原型基于既有的 Android 产品设计文档与实现计划，只覆盖关键体验路径，不实现完整课程逻辑，不引入真实后端或题库引擎。

原型目标不是“可上线的网页产品”，而是为后续平板端实现确认以下事项：

- 横版平板上的主体视觉语言是否成立
- 主流程结构是否顺畅
- 地图、关卡、答题、奖励是否在同一套世界观中统一表达
- 绘本冒险风格是否适合该学习产品

## 2. 原型范围

### 2.1 包含范围

原型包含以下 6 个关键状态：

1. `欢迎页`
2. `总海岛地图`
3. `岛屿关卡浮层`
4. `答题页`
5. `奖励结算`
6. `宝箱收藏`

### 2.2 不包含范围

原型不包含以下内容：

- 完整课程数据加载
- 全部岛屿与单元细节
- 真正的做题判分逻辑
- 家长端原型
- 响应式适配到手机尺寸
- 音效、语音、复杂粒子特效

## 3. 目标设备与画布约束

### 3.1 目标设备

- 主目标为 `横版平板`
- 以 `16:10` 比例为优先参考
- 默认视窗可按 `1280 x 800` 或相近比例展示

### 3.2 布局原则

- 始终以“横向大场景”作为核心舞台
- 避免竖版手机思维直接横置
- 信息面板应以浮层、抽屉、卷轴、木牌等形式叠加在地图之上
- 地图必须是主视觉，不允许被普通后台式界面替代

## 4. 视觉方向

### 4.1 总体风格

原型采用 `绘本冒险` 风格，强调儿童探险绘本与地图书的气质，而不是常规教育 App 的白底卡片界面，也不是夸张手游风。

关键词如下：

- `绘本感`
- `海岛探险`
- `纸感材质`
- `柔和暖光`
- `手工地图`
- `轻游戏反馈`

### 4.2 材质语言

主要视觉材质如下：

- `纸张`：用于背景层、卷轴、任务页
- `木牌`：用于主按钮、关卡入口、标题牌
- `贴纸`：用于宝箱收藏、奖励展示
- `柔光阴影`：用于层级与悬浮感

避免以下风格：

- 科技感玻璃拟态
- 赛博霓虹
- 电商式大白卡
- 电竞 UI 风格强发光

### 4.3 配色系统

建议使用暖色绘本海图配色：

- `海水蓝`：主背景与海面
- `沙滩米`：纸张、卷轴、浅底色
- `橘金色`：奖励、强调、星星
- `森林绿`：岛屿与安全感视觉
- `木棕色`：按钮、牌匾与结构边框

岛屿的主题色差异保留，但第一版原型只重点展示 `计算岛`，因此不需要一次把所有岛完整展开。

### 4.4 字体方向

字体表达目标如下：

- 标题：略有童趣、带故事感
- 正文：圆润、清晰、便于儿童阅读
- 数字：清楚、稳定、适合题目呈现

网页原型允许使用近似的 Web 字体组合模拟这种气质，但不得退化为普通企业后台风字体系统。

## 5. 信息架构与流程

### 5.1 主流程

原型覆盖以下主路径：

`欢迎页` -> `总海岛地图` -> `计算岛关卡浮层` -> `答题页` -> `奖励结算` -> `宝箱收藏`

并提供回退路径：

- 从关卡浮层返回地图
- 从奖励页返回地图
- 从宝箱页返回地图

### 5.2 原型策略

采用 `单页面伪交互原型`，在一个网页中模拟多个场景状态切换，而不是多页面网站。这样更接近最终平板 App 的体验，也更利于确认空间连续性与视觉统一性。

## 6. 页面与状态设计

### 6.1 欢迎页

目的：

- 建立世界观与品牌气质
- 让用户立即感受到这不是传统题库应用

必须包含：

- 标题 `Math Island / 数学岛`
- 一句简短冒险式引导语
- 主角色或探险伙伴氛围元素
- 大按钮 `开始冒险`

视觉要求：

- 具有绘本封面感
- 不堆功能入口
- 保留大面积场景插画空间

### 6.2 总海岛地图

目的：

- 展示横版平板的核心大舞台
- 验证地图是否足以承担主导航

必须包含：

- 横向展开的海岛地图
- 至少 4 到 5 个可见区域元素，例如海湾、桥、灯塔、树林、宝箱点位
- 可点击的 `计算岛`
- 浮在地图上的少量入口，例如 `继续冒险`、`宝箱`

视觉要求：

- 地图必须比按钮更抢眼
- 信息层不要遮挡主要路径和岛屿轮廓
- 视线应自然从左向右或从前景向岛屿深入

### 6.3 岛屿关卡浮层

目的：

- 展示进入岛屿后的任务结构
- 验证“地图为底，任务为浮层”的层级方式

必须包含：

- 关卡名
- 当前任务目标
- 2 到 3 个节点或小关展示
- `开始挑战` 按钮
- 关闭或返回地图入口

视觉要求：

- 浮层像木牌或任务纸页，不像普通弹窗
- 地图背景仍然保持可见

### 6.4 答题页

目的：

- 验证题目界面是否仍保持世界感
- 验证题目区和互动区是否适合横版平板

必须包含：

- 当前题目标题或任务目标
- 1 道示例题
- 简单交互，例如选择答案或数字输入
- `提交答案` 按钮

视觉要求：

- 题目容器像卷轴或任务书
- 背景仍属于岛内场景，而不是切成纯白练习页
- 界面上的文字量控制克制

### 6.5 奖励结算

目的：

- 确认即时反馈和游戏化激励的表达方式

必须包含：

- 星星结果
- 修桥/修复进度表现
- 关卡完成提示
- `打开宝箱` 或 `返回地图`

视觉要求：

- 节奏轻快但不刺眼
- 奖励信息一眼可懂
- 地图修复反馈要可见

### 6.6 宝箱收藏

目的：

- 验证奖励系统的视觉语言

必须包含：

- 宝箱开启后的贴纸或收藏物
- 至少 3 个收藏位展示
- 返回地图按钮

视觉要求：

- 偏贴纸册或收藏页气质
- 保持同一世界观，不切成独立商城风

## 7. 交互原则

### 7.1 交互形式

原型只做轻量伪交互：

- 点击按钮切换状态
- 点击岛屿打开关卡浮层
- 点击关卡开始挑战
- 点击提交进入奖励
- 点击宝箱进入收藏

### 7.2 交互要求

- 所有主操作必须是大触控目标，适合平板点击
- 动效以淡入、滑入、轻微缩放为主
- 页面切换要保留场景连续性
- 不允许使用 hover 作为核心交互前提

## 8. 视觉系统组件

原型至少需要以下组件：

- `主标题牌`
- `木牌按钮`
- `地图岛屿热点`
- `任务浮层`
- `卷轴题卡`
- `奖励星星组件`
- `宝箱卡片`

组件需共享统一的边角、阴影、描边和色彩规则，不能每个模块像不同产品。

## 9. 动效与反馈

原型中的动效应服务于情绪和层级：

- 页面进入：淡入 + 轻微上浮
- 按钮点击：轻微缩放反馈
- 浮层打开：柔和滑入
- 奖励出现：星星弹跳 + 柔光

避免：

- 高速闪烁
- 大范围粒子爆炸
- 过量位移动画

## 10. 原型验收标准

当原型完成后，应满足以下验收点：

- 用户能在网页中清晰走完主流程
- 第一眼能感知 `绘本冒险` 而不是普通学习工具
- 横版平板布局看起来自然，不像竖版页面硬拉宽
- 地图是主体而不是背景装饰
- 页面之间风格连续，没有割裂感
- 主要按钮与点击区域适合平板触控

## 11. 实施建议

建议原型使用基础 Web 技术完成：

- `HTML`
- `CSS`
- `JavaScript`

实现方式以单页状态切换为主，方便直接在浏览器打开预览，也方便后续快速调整风格。

## 12. 结论

该原型应优先完成 `横版平板 + 沉浸式全景地图 + 绘本冒险` 这三项核心验证。只要这三项成立，后续再向真实课程内容、更多岛屿、完整交互迁移才有意义。
