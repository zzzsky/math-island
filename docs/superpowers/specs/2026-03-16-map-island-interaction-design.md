# 数学岛地图与岛屿交互体验设计

## 1. 概述

本设计是对现有 `Android tablet MVP` 的交互增强补充，目标是在不重写课程状态机、内容资产和奖励逻辑的前提下，补齐 `地图探索 -> 岛屿查看 -> 课程进入 -> 通关回流反馈` 这一段核心体验。

本设计服务于以下已有文档：

- `docs/superpowers/specs/2026-03-13-math-island-android-design.md`
- `docs/superpowers/plans/2026-03-13-math-island-android-tablet-implementation.md`
- `docs/superpowers/plans/2026-03-13-math-island-implementation.md`

本次范围仅覆盖 `地图/岛屿交互体验`，不扩展新的课程内容，不新增复杂美术流水线，不重做通用题型系统。

## 2. 设计目标

- 让地图从“关卡列表”升级为“可探索的主世界入口”。
- 让孩子明确感知“我当前在哪座岛”“下一步去哪”“刚刚推进了什么”。
- 保持现有课程与奖励逻辑稳定，仅在表现层和地图交互层增强体验。
- 为后续接入正式美术资源和动画资源预留组件边界。

## 3. 非目标

- 不在这一轮引入自由拖拽平移、缩放和复杂自定义画布物理。
- 不在这一轮引入真实生产级 Spine/Lottie 资源流水线。
- 不更改课程解锁规则、评分规则、复习规则和 challenge 规则。
- 不将地图做成完全开放世界；仍保留清晰的岛屿顺序与推荐路径。

## 4. 用户体验目标

### 4.1 核心体验

孩子进入地图后，能立刻回答三个问题：

- 我现在最适合玩哪座岛？
- 这座岛上有哪些课？
- 我刚刚完成后发生了什么变化？

### 4.2 体验原则

- 地图负责“探索与定位”
- 岛屿面板负责“决策与进入”
- 通关回流负责“反馈与奖励”

三者不混合，避免地图点一下就直接跳关，导致孩子失去方向感。

## 5. 组件边界

### 5.1 MapSceneCanvas

职责：

- 绘制地图背景、岛屿节点、航线、焦点高亮和状态徽记
- 响应岛屿点击
- 表达 `locked / unlocked / focused / completed` 四态

不负责：

- 课程列表渲染
- 奖励文案
- 业务解锁计算

### 5.2 IslandDetailPanel

职责：

- 展示当前焦点岛的标题、副标题、描述、整体进度
- 展示课程卡片列表
- 提供推荐课程 CTA
- 表达课程状态：`未开始 / 可开始 / 已完成 / 新解锁`

不负责：

- 地图绘制
- 课程结算

### 5.3 MapProgressFeedback

职责：

- 在课程完成回到地图时展示一次性反馈
- 表达 `新岛解锁 / 新课解锁 / 星星增加 / 宝箱可查看`
- 控制反馈只消费一次，不重复播放

不负责：

- 真正修改进度
- 长期存储反馈历史

### 5.4 MapInteractionViewModel

职责：

- 基于现有 `MapViewModel` 和 use case 输出地图交互所需的 UI 状态
- 决定默认焦点岛、推荐课程和短时反馈事件

不负责：

- 课程数据资产解析
- 课程结算状态机

## 6. 状态模型

在现有 `MapTabletUiState` 基础上补充一层交互态。

### 6.1 selectedIslandId

- 当前焦点岛屿
- 默认优先落在“推荐下一步”的岛
- 若无推荐，则落在第一座已解锁未完成岛

### 6.2 recommendedLessonId

优先级如下：

1. `pending review`
2. 当前焦点岛第一节未完成课
3. 当前焦点岛第一节已完成课的再次练习

### 6.3 recentUnlock

一次性的“最近解锁对象”，类型仅保留：

- `island`
- `lesson`

它不是长期业务状态，只用于本次回到地图时的反馈展示。

### 6.4 mapFeedback

一次性地图事件，建议最小集合如下：

- `new-route`
- `new-island`
- `new-lesson`
- `stars-earned`
- `chest-ready`

这些事件应在 UI 消费后清空，避免重复触发。

## 7. 交互规则

### 7.1 地图阶段

- 点击左侧岛屿节点：仅切换焦点
- 焦点岛应显示更强的轮廓、标题或脉冲态
- 未解锁岛允许展示，但点击后只显示“等待前岛完成”的弱交互反馈

### 7.2 岛屿面板阶段

- 右侧始终展示当前焦点岛详情
- 面板主按钮进入 `recommendedLessonId`
- 点击课程卡才直接进入对应课程
- 切换岛屿不自动进课

### 7.3 通关回地图阶段

- 如果新岛解锁：地图自动切到新岛焦点，并触发航线亮起/节点脉冲
- 如果只解锁新课：保留当前岛焦点，高亮新课卡片
- 如果拿到贴纸或宝箱奖励：宝箱入口显示一次强调提示
- 如果星星增长：地图顶部星星统计显示一次跳变反馈

## 8. 视觉与动效策略

本轮不依赖正式美术资源，先使用可替换的 Compose 表现层占位：

- 地图背景：渐变海面 + 柔和地形块
- 岛屿节点：分层圆角卡块或有机轮廓块
- 航线：贝塞尔弧线或点状路径
- 焦点：描边、发光、轻微缩放
- 完成态：旗帜/星章/灯塔亮起
- 新宝箱提示：按钮轻微呼吸或闪光边框

动效原则：

- 只做短反馈，不阻塞操作
- 进入地图后的反馈时长应控制在 300ms 到 900ms 之间
- 所有反馈都应可中断，用户点击岛屿后立即让位给新交互

## 9. 资源接入预留

为后续引入美术和动画资源，当前设计要求：

- `MapSceneCanvas` 将节点样式、航线样式、背景样式集中在单独的绘制层
- `MapProgressFeedback` 将一次性反馈表达为可替换的 visual token
- 岛屿插图、装饰层、动画资源都通过模型字段或资源映射表注入，不散落到业务逻辑中

这样后续可以逐步引入：

- 矢量插画
- Lottie 小动画
- 音效/Haptic 对应点

而不需要重写课程状态机。

## 10. 测试策略

### 10.1 地图节点交互

- 点击岛屿节点切换焦点
- 默认焦点落在推荐岛
- 已完成岛显示完成态

### 10.2 岛屿详情面板

- 标题、说明、进度和课程列表正确绑定当前焦点岛
- 主 CTA 指向推荐课程
- 点击课程卡进入 lesson

### 10.3 推进反馈

- 新岛解锁时触发一次性地图反馈
- 新宝箱时高亮宝箱入口
- 反馈消费一次后再次进入地图不重复出现

## 11. 并行实现建议

建议拆成三条并行任务：

1. 地图节点交互
2. 岛屿详情面板
3. 通关回流反馈

三条线共享现有业务状态，但只通过 `MapInteractionViewModel` 汇合，避免多处直接改 controller。

## 12. 验收标准

满足以下条件即可视为本轮完成：

- 地图上有可点击岛屿节点和明确的焦点态
- 右侧存在稳定的岛屿详情面板与课程入口
- 课程完成返回地图时，至少能正确展示一种新解锁反馈和一种奖励反馈
- 相关 Compose UI tests 和 device flow tests 通过
