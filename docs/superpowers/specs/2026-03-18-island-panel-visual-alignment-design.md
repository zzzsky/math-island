# 数学岛岛屿详情面板视觉统一设计

## 1. 概述

本设计聚焦地图右侧的岛屿详情面板与课程卡，把它们接入已经落地的地图视觉语言、岛屿 icon 资源链路和统一的 token 体系。目标是让地图本体、岛屿标题区和课程卡在视觉上成为一套完整系统，同时保持现有交互、课程入口合同和业务状态不变。

本轮不改动地图焦点切换逻辑、不改变课程解锁规则、不新增业务状态。只统一表现层、组件边界和资源复用方式。

## 2. 设计目标

- 岛屿详情面板的标题区复用地图同一套 island icon 资源。
- 课程卡和 CTA 与地图整体视觉语言统一。
- `panel-start-*` 入口合同保持稳定。
- 面板样式不再散落在 `IslandOverlaySheet` 单文件里，而是收成可复用组件。

## 3. 非目标

- 不修改课程进入规则。
- 不新增岛屿详情专属插画资源槽位。
- 不改家长页、奖励页和课中 renderer。
- 不在本轮重做整体 layout 或导航。

## 4. 视觉方向

延续地图当前的“轻手绘冒险感”，但让右侧面板从纯信息面板升级成“岛屿故事卡 + 课程任务卡”的视觉：

- 标题区：使用岛屿 icon、主题色、纸面或木牌质感。
- 描述区：保留故事说明，但放进更清晰的卡片容器。
- 课程卡：每节课像任务卡片，而不是普通按钮列表项。
- CTA：继续保持明显可点，但风格跟地图一致，不另起一套按钮语言。

## 5. 组件拆分

### 5.1 IslandPanelHeader

负责：

- 岛屿 icon
- 标题
- 副标题
- 进度文案或进度条

资源来源：

- 直接复用 `island_<id>_icon` 资源链路

约束：

- 不承载课程列表
- 不负责 CTA
- 不新增业务判断，只展示已经计算好的状态

### 5.2 IslandStoryCard

负责：

- 岛屿描述
- 当前探索提示或故事文案容器

约束：

- 只做内容容器，不加额外动作
- 不在此组件内放课程入口按钮

### 5.3 IslandLessonCard

负责：

- 课程标题
- 课程摘要
- 课程状态标签
- 进入课程 CTA

约束：

- 继续使用 `panel-start-<lessonId>` 作为稳定入口标签
- 允许显示 `未开始 / 可进入 / 已完成 / 新解锁`
- 不改变点击行为

### 5.4 IslandPanelTokens

负责：

- 标题区颜色
- 课程卡描边、底色、状态色
- 面板内部间距和卡片圆角

约束：

- 尽量复用现有地图 token 的色彩方向
- 但不把所有面板细节都塞进 `MapIllustrationTokens`

## 6. 资源复用策略

本轮资源复用只做一层：岛屿 icon 直接复用地图已接入的 canonical slot。

具体规则：

- `IslandPanelHeader` 通过现有 `MapArtRegistry.resolveIslandArt(islandId)` 获取 icon slot
- 如果 drawable 命中，则走资源
- 如果缺失，则继续回退到当前 painter fallback

课程卡不新增资源槽位，先通过 token、描边、状态色和共享组件完成统一。这样可以避免在没有正式美术资源时过早扩张 slot 数量。

## 7. 集成边界

`IslandOverlaySheet` 继续作为岛屿详情面板的集成入口，但应该收窄职责：

- 它负责布局编排和动作回传
- 不再自己持有大量视觉细节
- header、story、lesson card 的具体样式都交给独立组件

这样后续要继续接面板插画资源时，也可以只替换 header 或 story card，而不用重写整个 overlay。

## 8. 测试与合同

本轮必须保持以下合同不变：

- `panel-start-<lessonId>`
- 地图点击岛屿只切换焦点，不直接开课
- 当前选中岛切换后，右侧面板内容同步更新

需要补强的测试：

- `IslandOverlaySheet` 级别测试
  - 标题区能显示 icon、标题和进度
  - 课程卡能显示状态和 CTA
- `MapTabletScreenTest`
  - 选择不同岛后，右侧面板继续更新
  - `panel-start-*` 数量和显示不变
- `MathIslandTabletFlowTest`
  - 主线进入课程、解锁新岛、回地图后不回归

## 9. 验收标准

- 岛屿标题区、地图节点 icon、右侧面板风格显著统一
- 课程卡风格与地图一致，不再像普通列表按钮
- `panel-start-*` 合同稳定
- 岛屿切换和课程进入行为不变
- 新组件边界清晰，`IslandOverlaySheet` 不再是视觉细节堆积点
