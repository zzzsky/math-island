# Return Stage Final Copy Design

## Goal

Compress reward, map-return, island-handoff, and map-list return copy into one shorter final language system.

## Scope

- tighten `MapReturnCopy`
- keep four return kinds distinct
- align reward/map/island/list consumers to the same wording

## Copy Rules

- summary titles stay short and decisive
- detail titles become the shortest “what to do next” line
- action titles remain the strongest imperative
- list bodies become one short secondary line

## Kind Targets

- `NewIsland`: emphasize mainline momentum
- `Chest`: emphasize collection first
- `Replay`: emphasize review first
- `Progress`: emphasize direct continuation

## Non-goals

- no structural UI changes
- no new motion changes
- no route or CTA behavior changes
