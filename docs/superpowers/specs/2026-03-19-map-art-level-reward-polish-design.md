# 地图占位美术与课中结算页视觉抛光设计

## 1. 概述

本轮设计同时推进两条视觉抛光主线：

- 地图 placeholder 资源继续往“更像正式绘本占位”推进
- 课中页与奖励页提升视觉完成度，但保持比地图更干净的产品感

目标是在不修改地图交互、题目逻辑、挑战规则和稳定测试合同的前提下，让地图世界观表现更完整，同时让主学习流程的课中和结算页拥有与地图页同级别的视觉质量。

## 2. 设计目标

- 地图海面、岛屿 base、航线和 icon 的占位资源更像一版草绘定稿，而不是纯技术占位。
- `LevelTabletScreen` 保持信息清晰，但加入少量绘本元素，和地图页形成同一产品语言。
- `RewardOverlay` 更像一张结构清楚的成绩单，而不是单块深色面板。
- 所有现有稳定 tag、业务状态、课程入口和挑战流保持不变。

## 3. 非目标

- 不改动地图 slot key、资源解析优先级和 `MapArtRegistry` 的公共接口。
- 不改动题型 renderer、题目内容和挑战结算规则。
- 不新增正式美术资源流程，只继续增强默认占位资源。
- 不在这轮继续扩张地图动效。

## 4. 视觉方向

### 4.1 地图

地图继续采用“轻手绘冒险绘本”方向，但把占位资源做得更完整：

- 海面更有纸感和轻微纹理层次
- 岛屿 base 更有主题轮廓和装饰
- 航线更像手绘刷线或虚线
- icon 更贴近岛屿主题

### 4.2 课中与奖励

课中页和奖励页保持“更干净的产品界面”，只借用少量绘本元素：

- 使用统一的 surface/card 层级
- 标题 chip、分组标题、角标或轻装饰线保留少量绘本感
- 避免大面积重纹理和过多装饰，保证信息优先

## 5. 地图资源抛光

### 5.1 海面背景

- 继续使用 `map_sea_backdrop`
- 强化纸面底纹、淡波纹和轻微色带变化
- 仍需保持文字和节点可读性，不可过亮或过花

### 5.2 岛屿 base

每座岛的 `island_<islandId>_base` 应具备：

- 更不规则的岛屿轮廓
- 与主题相关的地表分区
- 1 到 2 个轻装饰物，例如桥、尺子、旗帜、路径、山丘、木牌

目标是让 7 座岛在未点开之前也有更强的辨识度。

### 5.3 航线

- `route_segment_default` 继续保持清晰可读
- `route_segment_highlight` 在高亮时更像暖色扫线
- 仍需支持任意路径铺设，不引入复杂位图拼接

### 5.4 岛屿 icon

`island_<islandId>_icon` 应更明确地表达主题，例如：

- calculation: 算盘/数字木牌
- measurement: 尺子/三角尺
- multiplication: 旗帜或口诀木牌
- division: 分组木桶或分堆箱
- big number: 望远镜或数字卷轴
- classification: 收藏箱或分类板
- challenge: 罗盘或冠军旗

## 6. 课中页抛光

### 6.1 总体结构

`LevelTabletScreen` 继续保持左右分栏：

- 左侧为信息和流程区
- 右侧为答题区

本轮只抛光左侧信息区的视觉层次，不改右侧答题 renderer 逻辑。

### 6.2 视觉处理

- 课中主卡继续用 page surface，但增加更清晰的分组感
- 题目说明卡与流程提示卡使用 secondary surface，并统一标题节奏
- 计时信息、进度条和题号信息形成稳定顶部区
- 返回地图按钮和信息块之间的视觉节奏更清楚

### 6.3 绘本元素

只保留轻量绘本元素：

- 小型标题 chip
- 信息卡顶部的轻装饰线或强调色带
- 少量柔和描边和暖色点缀

## 7. 奖励页抛光

### 7.1 总体结构

`RewardOverlay` 目标是更像“成绩单 + 下一步建议”：

- 顶部结果摘要
- 中部成绩统计卡
- 新奖励 / 新岛屿 / 下一步卡
- 底部主次 CTA

### 7.2 视觉处理

- 主容器改成更清晰的 layered card，而不是单块深色板
- 统计卡形成统一的一组三联卡
- 成绩分级、新奖励和下一步卡有更明确层次
- 次按钮和主按钮在视觉上区分更清楚

### 7.3 绘本元素

仍然只保留少量装饰：

- 成绩分级卡的角标感
- 新奖励卡的小贴纸感
- 标题区少量暖色强调

## 8. 组件与文件边界

### 8.1 地图占位资源

- `app/src/main/res/drawable/*`
  更新现有 canonical map slot 的 vector drawable 占位资源
- `app/src/main/java/com/mathisland/app/ui/components/map/*`
  只做必要的 painter 适配，不改公共合同

### 8.2 课中页

- `app/src/main/java/com/mathisland/app/feature/level/LevelTabletScreen.kt`
  负责课中页信息区抛光
- 如需要新增小型局部组件，应放在 `feature/level/` 下，避免把地图 token 逻辑引入课中页

### 8.3 奖励页

- `app/src/main/java/com/mathisland/app/feature/level/RewardOverlay.kt`
  负责奖励页结构和卡片层级抛光
- 如需要新增局部结果卡组件，也应落在 `feature/level/`

## 9. 稳定合同

本轮不得改动以下合同：

- `panel-start-<lessonId>`
- `map-open-chest`
- `map-total-stars`
- 现有地图节点、航线高亮相关 tag
- 课中答题和奖励页返回主路径

## 10. 验收标准

- 地图占位资源明显更接近一版正式草绘，而不是简单技术占位
- 课中页与奖励页视觉完成度明显提升，但不影响信息清晰度
- 地图、课中、奖励三个主路径看起来属于同一产品系统
- 单测、instrumentation 和 assemble 在阶段收口时继续通过
